Time and speed

Your village runs on its own clock. Here's how game time maps to real time, how to speed it up or slow it down, and how pausing actually works.

The game clock

Every "tick" of the simulation is exactly one game second — the smallest unit of time the world tracks. The calendar starts at day 1, 06:00 (tick 0), and a full game day is 86,400 ticks. The clock never drifts or rounds oddly, so the game time you see displayed (day N HH:MM) stays exact no matter how long a world has been running.

The speed ladder

Speed controls how fast game time moves relative to real time. There are six settings:

SpeedMeaning
1xOne game second per real second
4x (default)One game minute per 15 real seconds
8xTwice as fast as 4x
16xTwice as fast as 8x
32xThe top of the "watchable" ladder — still fast enough to follow along, but the last stop before things get unreadable
maxAs fast as your machine can manage, uncapped — meant for worlds with no AI-driven villagers configured; it isn't offered once a model is set up, since decisions need at least a little real time to think

Change speed from the command line, or step through the ladder live from the full-screen client with the [ and ] keys (which cycle 1x → 4x → 8x → 16x → 32x; max is deliberately left off that quick-step ladder, reachable only by naming it directly):

promptworld speed demo 8x
promptworld speed demo max

Pause and resume are player verbs

Pausing is something you do on purpose — it's different from just detaching your view (which never stops the world). From the command line:

promptworld pause demo
promptworld resume demo

In the full-screen client, the space bar toggles pause/resume directly. When you pause, the world truly freezes: any villager mid-thought at that instant gets to finish thinking and land its decision at the exact frozen moment, as if no time had passed for it — nothing new starts until you resume.

Your speed is a ceiling, not a promise

If you've got AI-driven villagers, the speed you set is the fastest the world will run — not a guarantee. When several villagers are thinking hard at once at a high speed, model replies can struggle to keep up with how fast game time is passing, and a decision that lands late risks being nonsense for the moment it was meant for. Rather than let that happen, the world automatically drops down one speed notch on its own until the backlog of in-flight thoughts clears, then climbs back up again — all the way back to the speed you actually asked for — once things catch up. You don't have to do anything; it just happens.

While this is going on, the full-screen client's header spells out exactly what's happening, for example asked 32x — 3 minds in flight, debt 140%: the speed you asked for, how many villager thoughts are currently in flight, and how overdue they are as a rough percentage. The chronicle's raw feed gets two matching line types for the same thing — a governor shed line when the world drops a notch, a governor recovered line when it climbs back — so you can always see when and why your effective speed changed, even after the fact.

A few things stay true no matter what:

  • Pausing still lets any villager mid-thought finish and land, exactly as before.
  • Your own speed commands always win instantly — asking for a new speed, or pausing, immediately overrides whatever the automatic throttle was doing.
  • max speed still flatly refuses to run once an AI model is configured; the automatic throttle only ever manages notches on the watchable 1x–32x ladder, it never invents a safe uncapped mode.

The world is always honest about its own pace

Separately from the throttle above — which is about giving villagers' minds enough real time to stay sensible — the world is also honest about raw throughput: if your machine can't actually keep up with the speed you asked for at all, the world doesn't try to catch up by skipping ahead — it slows down for real and tells you so, rather than silently jumping over events you would have wanted to see. There's no "fast-forward burst" to catch up on a backlog; what you watch is always what actually happened, at the pace it actually happened.
