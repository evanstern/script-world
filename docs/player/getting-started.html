Getting started

From "never seen this before" to a village running in your terminal — install, create a world, start it, and watch it live.

What promptworld is

promptworld is a terminal game on a procedurally generated map. You don't play a character — you populate a small village (8 villagers) and watch it run. Each villager has its own mind: some decisions come from a language model "thinking" for them, and simple things (walking, sleeping, eating) happen on the spot without any model call. The village is ambient and always-on: once you start a world, a background process (the "daemon") keeps simulating it around the clock, whether or not you're watching. You attach and detach like tuning into a channel — the world doesn't pause just because you looked away.

Before you start

You need Go installed to build the program. You do not need any AI model configured to get a world running — without one, villagers still walk around and handle their basic needs, just without model-driven decisions ("reflex-only" minds). If you do want AI-driven villagers, you'll want a local model server (an OpenAI-compatible endpoint, e.g. Ollama running something like gemma4:12b-mlx) and optionally an ANTHROPIC_API_KEY for the higher-quality cloud-narrated story feature. The AI setup basics page covers the minimum you need to flip that on; you can always come back to it.

1. Build the binary

go build ./cmd/promptworld

Everything — running the world, watching it, and managing it — is one program: promptworld.

2. Create a world

promptworld new demo --seed 42

This creates a new world named demo. A world is a self-contained save directory (by default under ~/.promptworld/worlds/demo) holding everything about that run — the map, the villagers, the history, and its AI configuration file. The --seed is a starting number for the world's random generation; the same seed always regenerates the identical map and starting conditions, so it's worth writing down if you want to recreate a world later. Leave it off and promptworld picks one for you.

Genesis also writes each villager's starting personality and Metatron's starting "charter" — more on both in a moment.

3. Start it

promptworld start demo

This starts the world's daemon in the background (detached — it survives you closing the terminal) and confirms it's up before handing control back to you. From this moment the world is running 24/7, ticking away even if you don't attach to it again until tomorrow. If something goes wrong, promptworld will point you at daemon.log inside the world's save directory.

You can check on any world, running or not, from anywhere:

promptworld ps        # every running world you have, at a glance
promptworld status demo   # this world's tick, game time, and speed

4. Watch it live

Two ways to watch, both non-destructive — quitting either one just detaches; the world keeps running underneath.

  • promptworld attach demo — a simple line-by-line feed of events as they happen, plus typed commands (pause, resume, speed <v>, status, quit).
  • promptworld ui demo — the full-screen visual client: a map of the village on one side, and a tabbed panel on the other for the chronicle (the narrated story so far), Metatron (your conversation with the village's angel — see the next page), and the villagers themselves (their needs, goals, and inventories). This is the recommended way to actually play.

In the full-screen client, space bar toggles pause/resume, and bracket keys ([ / ]) step you through the speed ladder. The time & speed page explains what those numbers mean.

5. When you're done for now

Quitting attach or ui only detaches your view — the daemon keeps the village running. If you actually want the world to stop simulating (e.g. you're shutting down your machine), stop the daemon itself:

promptworld stop demo

This is a graceful stop — it finishes its current bookkeeping before exiting. The world resumes exactly where it left off next time you start it, even if it was killed abruptly instead.

Quick reference

go build ./cmd/promptworld

promptworld new demo --seed 42     # create a world
promptworld start demo             # start the daemon (detached, runs 24/7)
promptworld ps                     # see every world you have running
promptworld status demo            # tick, game time, speed for one world
promptworld attach demo            # watch live, line-mode; quit just detaches
promptworld ui demo                # full-screen map + chronicle + Metatron + villagers
promptworld stop demo              # graceful stop; resumes cleanly next start
promptworld help                   # the full command list
